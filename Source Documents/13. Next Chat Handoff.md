## The key correction

Earlier prototypes v0.1 and v0.2 accidentally centered the loop on manual text input.

The old effective shape was:

```text
manual text input
→ prediction
→ signal
→ interpretation
→ evidence
→ belief update / no-update
→ action / no-action
→ safety
→ outcome
→ memory
```

That was useful as a learning spike.

But it is not the intended foundation.

The corrected rule is:

```text
model first
prediction second
signal third
prediction error fourth
```

Manual text is only one possible signal source.

It is not the root of cognition.

The old prototype began too close to manual text input.

```text

```

But the system should not add broad sources early.

It should be designed to support many sources while enabling only safe bounded sources at first.

## The corrected architecture

The corrected loop is:

```text
body/system state + priors + world model
→ prediction
→ multi-source signal intake
→ prediction comparison
→ prediction error
→ evidence quality
→ world-model / belief update or no-update
→ action / no-action
→ safety
→ outcome
→ memory
→ revised model
```

The corrected record order should be:

```text
BodyStateRecord
PriorModelRecord
WorldModelRecord
→ PredictionRecord
→ SignalSourceRecord
→ SignalRecord
→ PredictionErrorRecord
→ EvidenceRecord
→ ModelUpdateRecord or NoUpdateRecord
→ ActionProposalRecord or NoActionRecord
→ SafetyCheckRecord
→ OutcomeRecord
→ MemoryTraceRecord
→ LoopRecord
```

The order matters.

Tests should protect this order.

---

## The most important rule

```text
Prediction must be generated from body state, priors, and world model.
Prediction must not be copied from incoming manual text.
```

Bad pattern:

```text
manual text says X
→ prediction expects X
→ signal confirms X
```

Correct pattern:

```text
world model expects X
→ signal shows Y
→ compare X and Y
→ prediction error
→ evidence quality
→ update/no-update
```

This distinction is the heart of the project.


## Evidence policy

Signals are not truth.

Prediction error is not proof.

Evidence quality gates model update.

The system should preserve:

```text
NoUpdate
NoAction
Abstention
GatherMoreSignal
HumanReview
```

as valid outcomes.

A system that refuses weak updates and unsafe actions is behaving correctly.
